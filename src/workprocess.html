Arbetsprocess

Skapa Projektmapp och Initiera npm

Först skapade jag en ny mapp på min lokala dator för projektet. I den nya mappen öppnade jag terminalen och körde kommandot ”npm init”. Detta genererade en ”package.json”-fil i mappen, som innehåller metadata om projektet och hanterar dess beroenden. Därefter tog jag bort fälten "main" och "scripts" från ”package.json”-filen

Parcel installation

För att installera Parcel, som är ett utvecklingsverktyg för att bunta och optimera webbapplikationer, körde jag kommandot ”npm install parcel --save-dev”, som skapade en ”node_modules”-mapp som innehåller alla nödvändiga beroenden.

Uppdatera package.json

I ”package.json”-filen lade jag till följande konfiguration (“source”: “src/index.html”, "scripts": {"start": "parcel", "build": "parcel build"}), för att definiera startsidan och skripten för att starta och bygga projektet. Jag skapade sedan en ”src”-mapp och placerade mina HTML-filer, och i mappen skapades egna undermappar för respektive CSS- samt JavaScript-fil.

Starta Lokal Liveserver

För att starta en lokal utvecklingsserver körde jag kommandot: ”npm run start” Detta startade en lokal liveserver som övervakar filändringar och automatiskt laddar om sidan i webbläsaren.

Skapa .gitignore-fil

Jag skapade en ”.gitignore”-fil för att exkludera onödiga filer och mappar från versionshantering med Git, är ”node_modules”, ”dist”, ”.parcel-cache”, och ”package-lock.json”.

Versionshantering med Git och GitHub

Jag initierade ett nytt Git remote-repository via Visual Studio Code. För att hantera utvecklingsflödet skapade jag en ny branch som heter ”development” och började skriva regelbundna commits för alla ändringar jag gjorde.

Struktur och Layout

Jag skapade en logisk struktur för webbplatsen och använde semantiska HTML-element såsom ”header”, ”main” och ”footer”. En navigeringsmeny skapades för att underlätta användarnas navigering på webbplatsen. Jag designade layouten för att vara responsiv och användarvänlig.

Undersida för Bilder

Jag skapade en undersida ”pictures.html” där jag inkluderade minst fem av mina egna fotografier. Bilderna, som var i JPG-format, placerades i picture-element för att automatiskt kunna bli optimerade till moderna bildformat som AVIF och WEBP.

Dokumentering av Arbetsprocess

Denna undersida skapades där jag dokumenterade hela arbetsprocessen, från planering till implementering och publicering. Detta inkluderar verktyg och tekniker jag använde.

Sammanfoga Brancher

När utvecklingen var klar sammanfogade jag ”development”-branchen med ”main”-branchen för att integrera alla ändringar.

Bygga för Produktion

Genom att köra kommandot:”npm run build” transpilerade jag modern JavaScript till en version som är kompatibel med äldre webbläsare. Koden minifierades för att minska filstorleken och förbättra laddningstiderna.

Publicering

Jag publicerade projektet på Netlify genom att importera repository från GitHub. Netlify hanterade distributionen och gjorde webbplatsen tillgänglig online.
